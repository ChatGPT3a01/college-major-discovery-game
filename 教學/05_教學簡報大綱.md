### 投影片 4-6：什麼是 RPG 遊戲？

**Slide 4: RPG 定義**
```
RPG (Role-Playing Game) = 角色扮演遊戲

特徵：
1️⃣ 玩家扮演一個角色
2️⃣ 通過選擇影響故事
3️⃣ 角色有屬性和進度
4️⃣ 多條故事路線

例子：
- 《尋夢環遊記》(選擇式冒險)
- 《火焰徽章》(戰術 RPG)
- 《文字冒險遊戲》(互動小說)
```

**Slide 5: 教育 RPG 的特殊性**
```
vs. 傳統 RPG

傳統 RPG：
- 戰鬥、冒險、升級
- 目標：完成故事線

教育 RPG：
- 故事、抉擇、屬性成長
- 目標：幫助玩家做出人生決定

🎓 本課程設計的遊戲：
幫助大學生找到適合的科系
透過與各領域導師的互動

每個選擇都代表一個人生決定！
```

## 📖 第三章：故事與角色設計（30 分鐘）

### 投影片 16-20：故事結構

**Slide 16: RPG 故事的三層次**
```
Level 1：主線故事
整個遊戲的大框架
例：「探索科系」

Level 2：導師故事線
每位導師的獨特故事
例：「環保工程師的冒險」

Level 3：選擇點
玩家的每個決定都影響故事
例：「冒險 vs. 保守方案」

🎮 結構：
Level 1
  ├─ Level 2 (導師 A)
  │   ├─ Level 3 (選擇 1)
  │   ├─ Level 3 (選擇 2)
  │   └─ Level 3 (選擇 3)
  ├─ Level 2 (導師 B)
  │   ├─ Level 3 (選擇 1)
  │   ├─ Level 3 (選擇 2)
  │   └─ Level 3 (選擇 3)
  ...
```

**Slide 17: 好的故事選擇的特徵**
```
✅ 好選擇：
- 反映不同的人格或價值觀
- 有真實的後果
- 沒有明顯的「正確答案」
- 影響最終結果

❌ 壞選擇：
- 都導向相同結果
- 無法區分
- 明顯一個好一個壞
- 無影響力

例子：

❌ 壞：
A. 努力學習 (+5 分析力)
B. 努力學習 (+4 分析力)

✅ 好：
A. 冒險嘗試新方案 (+5 創意力、+7 勇氣、-2 分析力)
B. 使用已驗證方案 (+5 分析力、+3 領導力、-1 創意力)
C. 先向導師學習 (+6 好奇心、+4 同理心)
```

**Slide 18: 屬性系統設計**
```
我們遊戲的 6 個屬性：

1. 創意力  🎨 想像、創新、藝術表現
2. 分析力  🧠 邏輯、問題解決、數據
3. 領導力  👑 決策、團隊、影響力
4. 同理心  💚 溝通、理解、共感
5. 勇氣    ⚡ 冒險、挑戰、毅力
6. 好奇心  🔍 探索、學習、創新

屬性與職業的映射：

創意力 + 分析力 = UI 設計師
領導力 + 同理心 = HR 主管
分析力 + 好奇心 = 數據科學家
...

設計原則：
不追求完美平衡，而是讓玩家發現自己的獨特組合！
```

**Slide 38: 常見問題與解決**
```
問題 A：「故事太短了」
解決：增加更多故事節點

問題 B：「選擇感不強，結果都一樣」
解決：讓不同選擇產生不同後續故事

問題 C：「屬性加成不明顯」
解決：在選擇後立即顯示屬性變化

問題 D：「想要重新玩」
解決：加入「快速重玩」或「跳過已看故事」

問題 E：「結果太籠統」
解決：改進 AI 提示詞，生成更個性化結果

改進優先級：
1. 致命 bug （功能不正常）
2. 內容問題 （故事不吸引人）
3. UI 問題 （使用不順暢）
4. 美化 （視覺不夠漂亮）
```

**Slide 39: 效能優化**
```
常見優化項目：

【圖片優化】
✓ 壓縮圖片大小 （80% → 20% 可能）
✓ 使用 WebP 格式
✓ 懶加載 （用到才載入）

【代碼優化】
✓ 縮小化 minify
✓ 移除不必要的導入
✓ 使用 gzip 壓縮

【邏輯優化】
✓ 避免不必要的重新渲染
✓ 使用 memo 緩存組件
✓ 優化算法複雜度

【結果】
原本：首頁載入 5 秒
優化後：首頁載入 1 秒

用工具檢查：
- Lighthouse (Chrome)
- WebPageTest
- GTmetrix
```
